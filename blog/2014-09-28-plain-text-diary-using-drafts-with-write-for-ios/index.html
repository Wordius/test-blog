My simple plain text diary is created using Drafts and Write for iOS with x-callback-url

Keeping a diary is one thing I have never been good at.1 All those broken promises – to myself – whether during the days of pen and paper or, more recently, flirting with Day One for Mac, weigh heavily. It is something that I feel I should do, but, more often than not, don’t. Previous attempts have required too much effort. I also don’t like the way diary entries in Day One are locked in and only accessible by a single app. Day One does offer the option to export entries to plain text, PDF or Markdown, the latter only in the Mac app, but that’s an extra step. I ruled out Evernote for similar reasons. Proprietary file formats, such as those used by Evernote and Microsoft Word, stifle productivity and portability. So, if a future-proof, plain text diary is what’s called for, it makes sense to create it in plain text from the outset.

Dr Drang’s ‘Better work diary actions’ convinced me that this was the way forward, not backward, as some might think. Drang’s work diary actions combine Launch Center Pro (LCP) with Drafts to create and save a note per day in Dropbox. Each note can contain multiple entries. He uses LCP for ‘status updates’ and ‘check-ins’ – short notes, like ‘Left home’ or ‘Arrived at work’ – that he can add simply by tapping an icon in Launch Center Pro. Drang may have good reasons for having a note per day, but I prefer a single note – daily-diary.txt – with new entries prepended to the file. As well as using LCP status updates similar to Drang’s, I can add longer entries by typing directly in Drafts and invoking my ‘Diary entry’ action.

Before I get down to the how, it’s worth noting my reasons for keeping a diary:

  • To track back and find out what I was doing on any given day.
  • To record app purchases – as a reminder of how long I have been using an app.
  • To record product purchases – for warranties and the like.
  • To add my thoughts.
  • To record birthday and Christmas present purchases – so I don’t end up buying family members the same old dross time and again. This way they get a different kind of dross the following year.

iCloud instead of Dropbox #

As mentioned, Drang’s method uses Dropbox, but I’m swayed to use iCloud instead. So my Drafts’ action x-callbacks Write for iPhone and Write for iPad, both of which are set up to sync with iCloud. Write’s iOS apps will sync with iCloud or Dropbox, not both. So in making this choice, Write does not have access to my main ‘Notes’ folder, stored in Dropbox (at least it doesn’t on iOS). So what exactly is this doing? First we’re using Write’s url scheme, write app://. We then tell it to prepend the content of the current Drafts note, [[draft]], to a file called Diary-entry.txt. But before adding [[draft]], we insert the current date and time as a Markdown level 2 heading (##): %23 is url encoding for a hash symbol, %20 for a space. Then we insert two linefeeds, %0A%0A, followed by the note itself, [[draft]], and return to Drafts on successful completion of our entry. Note that I have not added any linefeeds at the end of the entry, as Write does this automatically.

Then on my Mac, I can access my diary in Write for Mac’s iCloud tab. Handily, Write allows me to access both my Write-only iCloud files, as well as my main ‘Notes’ folder in Dropbox and any other folder I choose to add to Write for Mac’s ‘Library’. I can open and edit daily-diary.txt in Write for Mac as I see fit. But I have semi-automated the process by creating a simple Keyboard Maestro macro, as shown in the screenshot below. Keyboard Maestro is a handy Mac productivity tool to have around. This particular macro does the following:

  • Locates and opens daily-diary.txt
  • Starts a new entry by adding a Markdown-formatted date and time
  • Puts the cursor on a blank line ready to accept a new entry.

Some explanation of how I built this Keyboard Maestro might come in handy here. Select This hot key from the toggle menu and assign a hot key in the field below; I chose ^⌘D. Type open in the Actions pane’s search field and double-click Open a File, Folder or Application to add that action to your workflow. In the field next to Open add the OS X text path2 to the daily-diary.txt file. Next to with, toggle Default application and choose Write from the list.

The next action in the workflow automatically adds the date and time for the diary entry, and inserts a line space after. Return to the Action pane’s search field, type Insert and double-click the Insert Text action to add it to your workflow. As you can see from the image above, I have added the date and time as a Markdown level 2 heading (##), just as I did in the Drafts’ action I use on iOS. You can type out the LongDate and ShortTime strings or insert them from the Insert Token drop down. Hit return twice and add the Position cursor token from the Insert Token drop down.

Advantages #

Keeping a diary in this way is much more efficient than any of my old ways. It’s always only a few keystrokes or taps away, which makes it increasingly likely I will keep a diary in the first place.

It also serves another purpose – check-ins. I’ve only ever dabbled with Foursquare and Facebook check-ins. But this is not the kind of information I feel the need to share with anyone else. Using Drang-style LCP status updates, I can keep a record of where I have been and what I was doing. These will serve as reminders should my future self want to find out what I was up to on any given day.
